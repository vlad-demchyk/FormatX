## Примітки щодо використання бібліотек

### pdf-lib
- Використовується для: копіювання PDF, merge/split PDF, створення PDF з тексту
- Обмеження: працює лише з WinAnsi кодуванням, потребує мапи Unicode→ASCII
- Не використовується для: PDF→DOCX (використовується pdf.js + docx)

### pdfjs-dist (версія 3.10.111)
- Worker завантажується з CDN: `unpkg.com/pdfjs-dist@3.10.111/build/pdf.worker.min.js`
- Використовується для: витягування тексту з PDF,
  конвертація PDF → TXT/HTML/MD/DOCX

### mammoth
- Легкий, не потребує WASM
- Конвертує DOCX → HTML (звідти через turndown → MD)
- Обмеження: базове форматування, не підтримує складні стилі

### marked
- Підтримує async парсинг
- Створює стилізований HTML з CSS для таблиць, код-блоків, зображень
- Використовується як проміжний шар для DOCX та PDF

### docx
- Створює документи з параграфами, TextRun, Header/Footer
- Підтримує PageNumber
- Використовується для MD→DOCX та PDF→DOCX

### xlsx (SheetJS)
- Читає XLSX, XLS, CSV
- Експортує в HTML таблиці, CSV, TXT, Markdown

### heic-to
- WASM модуль, потребує HTTP (не працює з file://)
- Підтримує вихідні формати: PNG, JPEG, WebP
- Quality: 0.1–1.0

### pdf-into-svg
- WASM модуль
- Ініціалізація при першому використанні (lazy)
- Повертає масив сторінок SVG
- Для багатосторінкових PDF об'єднує всі сторінки в один SVG

### sql.js
- WASM файл: `sql-wasm.wasm`
- Ініціалізується при старті додатку
- База зберігається в localStorage як base64-рядок
- При кожній зміні — експорт і збереження
